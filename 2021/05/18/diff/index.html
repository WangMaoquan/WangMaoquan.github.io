优化部分

使用 flags 作为 VNode 的标识

在vue2中, 判断是 VNode 还是 html 是这样来完成的

  • 拿到 VNode后先后尝试把它当做组件去处理, 如果成功的创建了组件, 说明该 VNode 就是该组件的 VNode
  • 如果没能成功地创建组件, 则检查 vnode.tag 是否有定义, 如果有定义则当作普通标签处理
  • 如果VNode.tag没有定义, 则检查是否为注释节点
  • 如果不是注释节点, 则当成文本节点

以上的操作我们只能在 mount 或者 patch 阶段 去完成, 换句话就是这是一个什么样的 VNode 只有在 挂载或者patch时才知道这是一个什么样的 VNode 所以 vue3 给每个VNode 一个 patchFlag 和 shapeFlag 字段 , 然后通过 VNode.shapeFlag & ShapeFlags.xxx 这样去判断是不是 相应的 类型, 通过 VNode.patchFlay & PatchFlags.xxx 去执行相应的patch 方法, 位运算的运行速度比 if 是更快的, 从而提高了性能

采用的双端比较

  • 使用 oldChildren 的第一个 VNode 与 newChildren 的第一个 VNode比较, 即 oldStartVNode 与 newStartVNode 比较
  • 使用 oldChildren 的最后一个 VNode 与 newChildren 的最后一个 VNode比较 即 oldEndVNode 与 newEndVNode 比较
  • 使用 oldChildren 的第一个 VNode 与 newChildren 的最后一个 VNode比较, 即 oldStartVNode 与 newEndVNode 比较
  • 使用 oldChildren 的最后一个 VNode 与 newChildren 的第一个 VNode比较 即 oldEndVNode 与 newStartVNode 比较
  • 如果上面四种都不满足, 先试图在 oldChildren 中寻找可以复用的 VNode
    1. 如果找到了 会把 oldChildren的那个可复用 VNode 赋为 undefined, 所以我们需要在 加两个判断, 遇到 oldChildren 中为 如果为 undefined 的时候 需要 ++ 或者 –
    2. 如果没有找到, 直接挂载当前的 newChildren[idxInOld]
