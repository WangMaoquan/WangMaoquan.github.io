总结

  1. 响应式WeakMap的存储为, 键为响应式对象(ref, reactive一堆的, computed), 值为一个effect Set集合 Set集合里面存储的就是这个响应式对象需要的依赖

  2. 响应式对象里面的 get 开始收集依赖 执行 track 方法, set 执行 触发依赖, computed不是通过通过自身的set去触发的, 通过回调里面的那个响应式对象 trigger后, 通过 computed的effect.options.scheduler方法, 执行 trigger

  3. effect的deps存放的是Set集合的数组

  4. computed effect 会被放在instance的effect数组里面

  5. track 方法里面 收集依赖的关键代码为

  6. trigger方法里面 触发依赖的关键代码
