vue中的生命周期

vue2的生命周期

beforeCreate, created, beforeMount, Mounted,beforUpdate, updated, beforeDestroy, destroyed, keep-alive组件特有的 activited, deactivited

beforeCreate: 初始化vm上的属性比如($parent, $root , $children, $refs, _watcher, _isMounted 等等), 初始化事件, 初始化render 比如(_c, $listeners, $attrs, $slots)

created: 初始化 inject, 初始化 props, data, methods, watch, computed, 初始化 provide

beforeMount: 获取el 的元素, 判断 传入的option, 是否有render, 没有render, 判断是否有 template, 有: 判断是字符串还是html 然后重新生成 template, 没有就获取 el元素的 outerHTML 作为template, 然后将 template 编译成 render方法, 实例上的$el 变成元素的el, 检查是否已经生成 render, 没有生成, 默认的为一个创建空的 VNode的方法 (createEmptyVNode), 开发环境下检查是否有 template 且 第一个字符不能为 “#” (这里检验使用的版本, 如果带有template 请使用runtime-compile, 或者指定render 方法)

mounted: 初始化 更新组件的方法, 初始化 更新组件的Watcher, 修改 组件是否挂载的标志(_isMounted)
destroyed: 修改标志(_isBeginDestroyed) , 存在 父级组件, 且父组件卸载, 卸载掉父组件中的自己, 卸载掉相关的watcher(同时也清除相关的依赖), 修改标志(_isDestroyed), 卸载当前组件, (清除事件其实是在destroyed 之后, 还做了 使组件实例$el__vue__ 为null, 也使了 vm.$vnode.parent为null)

vue3的生命周期

setup, onBeforeMount, onMouted, onBeforeUpdate, onUpdated, onBeforeUnmount, onUnmouted. onErrorCaptured, onRenderTracked, onRenderTriggered, 如果你要在 vue3 中使用 optionApi, 去掉on 就好

官方给的setup 对标的是 beforeCreate, created 其实参照源码来看, beforeCreate, created 的执行时在 setup 之后的(所谓的对标, 其实是在判断是否有setup函数时, 有就先执行setup函数, 拿到结果, 然后再判断setup函数返回值是否是函数, 是作为实例的render, 不是函数判断是否是对象, 不是对象且返回的不是undefined, 给出警告, 然后执行 finishComponentSetup方法,这个方法里面会调用 applyOptions 这个方法, 就会触发beforeCreate, created 钩子, 没有setup, 就是直接执行的 finishComponentSetup 方法) , 相比于vue2, 多了一个新的钩子 onErrorCaptured 用来捕捉错误, onRenderTracked 用于追踪 setup 返回的 响应式变量的, 这些值触发 track 的时候就会 触发该方法, 参数有(effect, key, target, type), onRenderTrigger 用于追踪 setup 返回的响应式变量, 这些变量触发 trigger 时 会调用该方法, 参数有(effect, key, newValue,oldTarget, oldValue, target, type)

vue 中的通信

vue2 中 : props 与 $emit , $children 与 $parent, eventBus, provide/inject, $refs, vuex, $listeners/$attrs

vue3 中: props 与 emit 如果要是用 emit 记得从 setup 的第二个参数(ctx: { emit, slots, attrs, expose })中解构或者 直接 ctx.emit() , expose 这个方法 用于导出组件中的一个方法, 然后在父组件中使用ref 就可以调用 该方法, 由于 vue3 已经把 $on 移除, 如果非得使用, 可以引入 mitt 库, $parent 与 $children 由于setup 里面没有this, 所以无法使用, 但是你可以在 template 中使用, 同理 也可以使用$listeners/ $attrs 因为 最后生成的render 函数是可以访问到this的``, provide / inject, vuex4.0 使用的useStore()`

v-if / v-show

v-if : DOM节点是否存在

v-show: display: none/block , 这里可能带出 隐藏元素的方法

  • display: none; 元素不会存在于DOM中, 无法触发绑定的事件
  • visibility: hidden; 元素仍会在DOM中, 无法触发绑定的事件
  • opacity: 0; 元素仍存在DOM中, 可以触发绑定的事件
  • position: absolute; top: -111111px; 元素存在DOM中
  • z-index 设置层级, 仍在DOM中
  • clip/clip-path, 仍在DOM中, 无法触发绑定事件
  • transform: scale(0, 0); 仍存在DOM中, 无法触发绑定事件

如果需要来回切换显示/隐藏, 推荐使用 v-show

vue 响应式原理

vue2

遍历 data 函数返回对象/data对象中的属性, 执行 defineReactive 方法, 然后定义好getter/setter 执行 Object.defineProperty, getter 中 收集依赖, setter 中通知 watcher 派发更新

vue3

通过 Proxy 代理, getter 中使用 track 收集effect, setter 通过触发 trigger 执行对应保存的 effects , 调用, vue3 使用了 weakMap 保存 相应的隐射关系(原始对象 => 对应的Proxy实例), ( effect 通过闭包保存的回调函数, 触发 getter 然后执行 track 方法收集 相关的 effect 这里也有一个weakMap对应的隐射关系如下 原始对象 => depsMap, depsMap 存放的是 key => effect Set集合)

Object.defineProperty vs Proxy

Object.defineProperty

  • 无法监听数组的变化

  • 对象层数过深, 需要递归

  • 无法监听新增属性和删除属性

    1. Vue.set(target, key, value) 怎么实现的

      target 是数组 且 key是合法的 数字, 通过 数组的 splice 方法

      key 存在于 target(会查找原型链的), 但是不在 Object的原型链上, 修改对应的key的值为 val 返回 val

      target 不是响应式对象, 将 对应的key 赋值为 val, 返回val

      target 为响应式对象, 将 key 在 target 中定义为响应式, 然后 手动触发依赖更新

    2. Vue.delete(target, key)

      target 为数组, 且 key 为合法的数字, 通过 splice

      target 不是响应式对象, 如果key不在 target 原型上, 直接return

      使用 delete target[key]删除, 如果是 响应式对象, 手动触发更新

Proxy

  • 直接监听的是对象, 不是属性, 对于层数深的对象, 我们可以访问到的时候再响应式下一层
  • 可以直接监听数组的变化
  • 有13种拦截方法
  • 作为新标准将受到浏览器厂商重点持续的性能优化
  • 可以对 Map, Set 也实现代理
